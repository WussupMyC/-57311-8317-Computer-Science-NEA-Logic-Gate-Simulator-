About Simboard

   
            Simboard is a Logic Gate simulator built by Christopher Cupid for the A-Level Computer Science NEA project!

            Simboard offers a simple range of gates and objects for interaction to aim to produce an environment where people who 
            are just learning about Logic Gates can educate themselves further and take on the challenge of creating a simple circuit. 

            To create this simple web application, HTML (Hyper Text Markup Language) was used to construct the layout of the webpage, 
            and CSS (Cascading Style Sheets) was used to style said structure. Finally, JS (JavaScript) was used as the backend 
            handler of events and logic utilised within Simboard, from the element of interactivity, all the way up to the logic that
            each of the gates use.


How to use Simboard and Logic Gate objects

   
            To first utilise and learn about Simboard and Logic Gates, you must first understand what parts of the application do what.
            Below is an exemplar with each major area of Simboard highlighted and labelled:

        
            Appendix 1  -   The File Menu
                This UI Menu has the purpose of providing the user with innate controls over the interfact and project. 

            Appendix 2  -   The Object Menu
                This UI Menu has the purpose of providing the user with the objects that can be imported into the Workspace. 

            Appendix 3  -   The Workspace
                This UI Menu has the purpose of providing the user with the ability of interacting and connecting objects together
                to formulate a circuit. 

            Appendix 4  -   The Filter Button
                This UI Button, when clicked, will pop up a new menu in that overlays the Object Menu, and provides the end user with
                the opportunity to filter objects down to a category for efficiency and comfort. 

            Appendix 5  -   The Simulate Button
                This UI Button, when clicked, will play and pause the simulation of the Logic Gate circuit in the Workspace. When the 
                simulation is paused, it will display "Play" on text and on its icon. When the simulation is playing, it will display 
                "Pause" on its text and icon. By default, the simulation is paused when Simboard is loaded in the web browser. 
                
                When the Workspace is paused, the user can drag and drop Logic Gate objects around the workspace, yet when the Workspace
                is playing, the user will be able to interact with buttons to fire execution flow. 

            Appendix 6  -   The Save Button
                This UI Button, when clicked, will open up a mini overlaying menu in the Workspace area. It will prompt you whether you 
                would want to save "As Layout" or "As Project". 

                Click "As Layout" if you want to save the scaling and colouring of the menus and UI that you may have changed when 
                on the Customise menu. 

                Click "As Project" if you want to save your Logic Gate circuit.

How to customise and configure Simboard

            You can customise the layout of the Simboard Application by selecting the "Customise" button in the File Menu. 

        
            Then, you can select the colours that you want the corresponding UI elements to appear as.

        
            Click "Save".

        
            Click "As Layout"

        
            The JSON file should now save to your local disk. 

        
            To load the layout when you next visit the website, click "Load" in the File Menu.

        
            Click "Layout File".

        
            Select the file that you saved "As Layout" and press "Open".

        
            You can now work with your desired layout again!
